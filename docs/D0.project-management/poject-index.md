# プロジェクトインデックスページ
## このページの位置付け
* 本プロジェクトの「ホーム」となるページです。
* 初めてプロジェクトに参画するメンバーがこのページだけを起点として自律的に情報収集できる状態を目指します。

## プロジェクト概要
### プロジェクト基本情報
* 名称: 個人保険新契約システム構築プロジェクト
* 対象業務: Sample生命保険株式会社における個人保険（終身保険・定期保険・医療保険・がん保険等）の新契約手続き(設計書作成・申込受付・告知・引受査定・第一回保険料収納・契約成立(計上)・保険証券発行)
* 本番リリース予定: 2026年9月

## 会議体

| 会議体 | 参加者 | 目的 | 開催頻度 |
|---|---|---|---|
| 経営会議 | Project Owner、経営層、PM(報告者) | 重要意思決定事項(予算、スコープ変更、予備費使用、リリース可否等)の決裁 | 月次 + 臨時 |
| 部長会 | 関連部門長、PM | プロダクト要求仕様書等の主要成果物の承認、部門間調整 | 月次 |
| リリース判定会議 | CIO、CISO、コンプライアンス部長、PM、QAリーダー | 本番リリース可否の最終判定 | 2026年8月(1回) |
| プロジェクト定例 | PM、業務責任者、IT責任者、各分科会リーダー | 進捗・課題・リスクの共有、平時の意思決定 | 週次 |
| 業務分科会 | 業務分科会リーダー、業務担当、開発・QA代表 | 業務要件・UX要件の詳細化と合意形成 | 週次 |
| システム分科会 | システム分科会リーダー、開発リーダー、インフラ担当、セキュリティ担当 | 機能要件・非機能要件・アーキテクチャ・セキュリティ設計の議論 | 週次 |
| QA定例 | QAリーダー、開発リーダー、業務代表 | テスト計画・進捗・不具合状況の共有、品質リスクの評価 | 週次(テスト期間中) |
| 外部連携調整会議 | IT責任者、開発リーダー、既存システム(契約管理・収納・帳票発送)担当 | 連携先システムとのインターフェース仕様調整、結合テスト計画 | 隔週(2026年6月末のIF仕様確定まで毎週) |

## プロジェクト運営ルール
### コミュニケーション方針
* 意思決定: 各分科会/部門での話し合いで発生したすべての意思決定はPMに通知します。
* エスカレーション: PMは平時の意思決定を行い、PMが「重要」と判断した事項はProjectOwnerや経営レイヤー向けの会議体で共有・決裁されます。
* Slack運用: 既存のチャンネルルールを維持しつつ、役割ごとにチャンネルを分割します。(具体的なチャンネル名・管理者はTBD(責任者決定後に記載))
